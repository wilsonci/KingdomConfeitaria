IMPORTANTE - SENHA DE APP DO GMAIL:
------------------------------------
O Gmail pode não aceitar a senha normal da conta para aplicações de terceiros.
Se você receber erros de autenticação, será necessário gerar uma "Senha de App".

COMO GERAR UMA SENHA DE APP:
-----------------------------
1. Acesse: https://myaccount.google.com/apppasswords
   (Você pode precisar fazer login na conta kingdomconfeitaria@gmail.com)

2. Se a opção "Senhas de app" não aparecer, você precisa:
   - Ativar a verificação em duas etapas na sua conta Google
   - Acesse: https://myaccount.google.com/security
   - Ative "Verificação em duas etapas"

3. Depois de ativar a verificação em duas etapas:
   - Volte para: https://myaccount.google.com/apppasswords
   - Selecione "Aplicativo": "Outro (nome personalizado)"
   - Digite: "Kingdom Confeitaria"
   - Clique em "Gerar"

4. Uma senha de 16 caracteres será gerada (exemplo: abcd efgh ijkl mnop)
   - Copie essa senha (sem espaços)
   - Substitua o valor de "SmtpPassword" no web.config por essa senha

5. Exemplo no web.config:
   <add key="SmtpPassword" value="abcdefghijklmnop" />

SEGURANÇA:
----------
A senha está armazenada no web.config em texto plano. Para maior segurança:

1. Em produção, considere usar criptografia de seções do web.config:
   - Use aspnet_regiis.exe para criptografar a seção appSettings
   - Comando: aspnet_regiis -pef "appSettings" "C:\Caminho\Para\Aplicacao"

2. Ou use variáveis de ambiente do servidor

3. Ou use Azure Key Vault ou similar para armazenar credenciais

TESTE DE ENVIO:
---------------
Para testar se o email está funcionando:
1. Faça uma reserva na aplicação
2. Verifique se o email de confirmação foi enviado
3. Verifique os logs de debug para erros

ERROS COMUNS:
-------------
1. "Authentication Required" ou "5.7.0 Authentication Required"
   -> Solução: Use uma Senha de App (veja instruções acima)

2. "The SMTP server requires a secure connection"
   -> Solução: Verifique se EnableSsl = true (já está configurado)

3. "Invalid username or password"
   -> Solução: Verifique se o email e senha estão corretos no web.config

4. "Connection timeout"
   -> Solução: Verifique se a porta 587 não está bloqueada pelo firewall

CONTATO:
--------
Se tiver problemas, verifique:
- Os logs de debug da aplicação
- A configuração do firewall
- Se a verificação em duas etapas está ativada
- Se a Senha de App foi gerada corretamente
